# Wubby Stream Archiver

A Docker-based system that automatically records [PaymoneyWubby](https://twitch.tv/paymoneywubby) streams from Twitch and Kick, generates visual contact sheets for browsing, and organizes recordings as a Jellyfin-compatible TV show library.

## Overview

When a stream goes live, the recording containers wake up and capture it. Once the stream ends, a post-processing pipeline runs automatically: contact sheet images are generated (grids of video thumbnails), a 4K poster image is created for each recording, and the files are renamed into episodic format (`S01E###`) so Jellyfin picks them up cleanly. The contact sheets are served via a Caddy web server for quick visual browsing.

## How It Works

```
Stream goes live
      │
      ▼
streamlink captures stream → ffmpeg writes MP4
      │
      ▼ (stream ends)
vcsi-trigger detects container exit
      │
      ▼
vcsi pipeline runs:
  1. Generate contact sheet (3×5 thumbnail grid)
  2. Generate 4K poster image (3840×2160)
  3. Rename + move to Jellyfin season directory (S01E###)
  4. Rename contact sheets to match
  5. Clean up orphaned sheets/posters
      │
      ▼
Caddy serves contact sheets at :8084
Jellyfin picks up new episodes automatically
```

## Services

| Container | Role |
|-----------|------|
| `streamlink-twitch` | Records Twitch streams continuously |
| `streamlink-kick` | Records Kick streams continuously |
| `contact-sheets` | Caddy web server for browsing contact sheets |
| `vcsi-trigger` | Watches Docker events and launches post-processing |
| `vcsi` | Generates contact sheets and Jellyfin posters (runs on-demand) |

## Contact Sheets

A contact sheet is a JPEG with a 3×5 grid of frames sampled evenly across the recording, with timestamps overlaid. They're generated by [vcsi](https://github.com/amietn/vcsi) and served at `http://<host>:8084/` for quick visual review of what happened in a stream.

## Jellyfin Integration

Recordings are organized as a TV show library:

```
TV Shows/
└── Wubby Streams/
    └── Season 01/
        ├── Wubby Streams - S01E01 - twitch-ab12cd34.mp4
        ├── Wubby Streams - S01E01 - twitch-ab12cd34.jpg   ← poster
        ├── Wubby Streams - S01E02 - kick-ef56gh78.mp4
        └── Wubby Streams - S01E02 - kick-ef56gh78.jpg
```

Poster images are 4K (3840×2160) so they look sharp at any display size. Episode numbers are assigned in chronological order and never reused.

## Recording Details

- **Codec:** stream copy (no re-encoding — fast and lossless)
- **Container:** MP4 with `+faststart` for seekability
- **Quality:** best available stream quality
- **Sentinel files:** a `.recording` marker file exists alongside each in-progress MP4 so the post-processing pipeline won't touch incomplete files

## Setup

### Prerequisites

- Docker and Docker Compose
- A Twitch OAuth token (for authenticated stream access)
- Healthchecks.io account (optional, for uptime monitoring)

### Configuration

Copy `.env.example` to `.env` and fill in the values:

```env
TWITCH_OAUTH=oauth:your_token_here
HC_UUID=your-healthchecks-uuid
HC_LOCAL_PING_URL=http://your-local-healthchecks/ping
VOD_DIR=/path/to/recordings
CONTACT_SHEETS_DIR=/path/to/contact-sheets
TWITCH_DIR=/path/to/twitch-root
```

### Running

```bash
docker compose up -d
```

### Manual Operations

Manually trigger contact sheet generation (with full rescan):

```bash
./manual-vcsi.sh
```

### Building Images

Images are hosted on GitHub Container Registry and rebuilt automatically on push via GitHub Actions. The server only needs `docker-compose.yml`, `.env`, and the mapped data volumes. To build locally:

```bash
# Streamlink recorder image
docker build -t streamlink .

# vcsi post-processor image
docker build -t vcsi ./vcsi

# Contact-sheet web image
docker build -t contact-sheets ./caddy

# Post-processing trigger image
docker build -t postproc-trigger ./postproc-trigger
```

## Health Monitoring

Each recording container pings [Healthchecks.io](https://healthchecks.io) when a stream starts and again when it ends (with the recorded duration). This makes it easy to get alerted if recordings silently stop working.
